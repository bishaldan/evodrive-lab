\section{Compared Methods}

\textbf{Genetic Algorithm.} EvoDrive Lab includes a custom genetic algorithm that evolves policy weights generation by generation. This is the clearest method for the project's live multi-car simulation because population turnover is directly visible in the interface and replay outputs.

\textbf{NEAT.} The benchmark also integrates \texttt{neat-python} as a topology-evolving baseline. This allows comparison between direct parameter evolution and evolving network structure.

\textbf{PPO-style baseline.} The repository contains a lightweight in-repo implementation of PPO-style policy optimization. This baseline is intentionally modest: it is included to provide a policy-gradient comparison point, not to stand in for a large production RL stack.

All three methods share the same environment, train/validation/test suites, and evaluation metrics.
