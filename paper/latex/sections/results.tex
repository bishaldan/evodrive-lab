\section{Results}

The paper reports validation and held-out test performance using completion, reward, crash rate, and mean episode length. The primary comparison focuses on whether the ranking of methods on validation tracks matches their behavior on unseen test tracks. In other words, the central question is not only which method performs best near the training distribution, but also which method degrades least when moved to held-out procedural tracks.

Figures generated by the paper plotting pipeline are intended to summarize:
\begin{itemize}[leftmargin=1.5em]
    \item validation versus test completion by algorithm,
    \item test reward by algorithm,
    \item test crash rate by algorithm,
    \item completion under sensor-count and track-length ablations.
\end{itemize}

The final manuscript should also include a small qualitative replay table that highlights one strong run, one median run, and one clear failure case per algorithm. These cases are selected from the aggregated run registry so that the qualitative discussion is tied to the same benchmark outputs as the quantitative tables.

When the benchmark matrix is complete, this section should be finalized in the following order:
\begin{enumerate}[leftmargin=1.5em]
    \item report the main comparison table from the aggregated CSV output,
    \item interpret validation-versus-test differences as the benchmark's generalization result,
    \item summarize the sensor-count ablation,
    \item summarize the track-length ablation,
    \item connect one qualitative replay example per method back to the numeric trends.
\end{enumerate}
